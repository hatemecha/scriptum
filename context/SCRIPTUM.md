## Documento Maestro del Proyecto

---

# 1. VISIÓN DEL PRODUCTO

SCRIPTUM es un software de escritura de guiones cinematográficos diseñado para ser:

- Simple
    
- Rápido
    
- Minimalista
    
- Profesional
    

Nace como respuesta a un problema claro:

> Las herramientas actuales de guion son arcaicas, sobrecargadas y poco amigables.

SCRIPTUM busca eliminar fricción y devolver el foco a lo esencial:

> ESCRIBIR.

---

# 2. PROBLEMA A RESOLVER

Los programas actuales:

- Son visualmente anticuados
    
- Están sobrecargados de funciones innecesarias
    
- Son incómodos para principiantes
    
- Interrumpen el flujo creativo
    

Muchos usuarios:

- No usan ni la mitad de las funciones
    
- Solo quieren escribir sin distracciones
    

---

# 3. PROPUESTA DE VALOR

> "Un software de guion profesional, simple, rápido y accesible para cualquiera."

SCRIPTUM debe ser:

- Tan simple como para principiantes
    
- Tan correcto como para profesionales
    

---

# 4. USUARIOS OBJETIVO

- Principiantes
    
- Estudiantes de cine
    
- Guionistas
    
- Escritores casuales
    

El sistema podrá adaptarse según el usuario:

- Detectar experiencia
    
- Ofrecer guía opcional
    
- Incluir glosario de términos cinematográficos
    

---

# 5. POSICIONAMIENTO

Si alguien compara SCRIPTUM:

> "Es como KIT Scenarist, pero más simple, moderno y cómodo."

---

# 6. FILOSOFÍA DEL PRODUCTO

SCRIPTUM es:

- Minimalista
    
- Claro
    
- Funcional
    

SCRIPTUM NO es:

- Una herramienta de producción
    
- Un software complejo
    
- Un sistema sobrecargado
    

---

# 7. EXPERIENCIA DE ESCRITURA

Inspiración directa:

- KIT Scenarist (flujo)
    
- Final Draft (formato)
    

### Características clave

- Flujo natural de escritura
    
- Uso intensivo de teclado
    
- Transiciones automáticas entre bloques
    

Ejemplo:

- Enter → siguiente bloque
    
- Tab → cambio de tipo
    

---

# 8. FORMATO

El formato será:

- Estrictamente profesional
    
- Basado en estándares de cine
    

Reglas:

- No se permite mezclar formatos
    
- No hay estilos libres
    
- Todo debe seguir reglas de guion cinematográfico
    

---

# 9. MODELO DEL DOCUMENTO

El documento se basa en bloques estructurados:

Tipos de bloque:

- Scene Heading
    
- Action
    
- Character
    
- Dialogue
    
- Parenthetical
    
- Transition

Referencia operativa cerrada del MVP:

- [SCREENPLAY_BLOCKS_V1.md](SCREENPLAY_BLOCKS_V1.md)
    

Ejemplo conceptual:

```
Scene
Action
Character
Dialogue
```

---

# 10. NAVEGACIÓN

El sistema incluirá:

- Sidebar de escenas
    
- Navegación rápida
    

---

# 11. MVP (PRIMERA VERSIÓN)

Debe incluir:

- Editor de guion funcional
    
- Formato automático
    
- Navegación básica
    
- Exportación a PDF profesional
    

Objetivo:

> Poder escribir un guion completo listo para mostrar.

---

# 12. FEATURES FUTURAS

No prioritarias:

- Colaboración en tiempo real
    
- Anotaciones
    
- Seguimiento de personajes
    

---

# 13. MODELO DE NEGOCIO

- Escritura completa

### Premium
